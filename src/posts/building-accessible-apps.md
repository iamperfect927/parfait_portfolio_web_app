# Building Accessible Web Apps

Accessibility (a11y) is not just a nice-to-have; it's a necessity.

## Semantic HTML

The foundation of an accessible web is semantic HTML. Use `<button>` for buttons, not `<div>`.

## ARIA Roles

When semantic HTML isn't enough, ARIA (Accessible Rich Internet Applications) roles bridge the gap.

> "The first rule of ARIA is: Don't use ARIA." - Use native elements whenever possible.

## Keyboard Navigation

Ensure all interactive elements are focusable and usable with a keyboard.
